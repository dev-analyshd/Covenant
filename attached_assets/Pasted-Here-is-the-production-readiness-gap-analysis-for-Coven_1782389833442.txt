Suggested Roadmap

Phase 1: Harden the Verifier (Weeks 1–8)
1. Implement full UltraHonk verification in `contracts/ultrahonk_verifier/src/lib.rs` using `bn254_pairing()` and the complete Protocol 26 host function set
2. Generate and verify 100+ proofs on testnet to catch edge cases
3. Benchmark gas costs — target < 0.50 per verification at current XLM prices

Phase 2: Proving Infrastructure (Weeks 4–12)
1. Build proving-as-a-service API: Client generates witness → sends to API → receives proof → submits to Stellar
2. Explore `bb.js` for browser proving if latency allows; fallback to server-side
3. Integrate with 1–2 KYC providers for live Merkle root feeds

Phase 3: Mainnet Pilot (Months 3–6)
1. Launch with 1 institutional partner (e.g., a regional remittance company)
2. Cap at Tier 1–2 (200K–400K limits) to limit blast radius
3. Run parallel with traditional compliance for 90 days to validate audit trails

Phase 4: Scale (Months 6–18)
1. Add recursive batch verification for cost amortization
2. Implement ASP logic for FATF-compliant privacy pools
3. Open KYC issuer network — charge for Merkle tree inclusion
4. Pursue regulatory sandbox approval in 1–2 jurisdictions

The Honest Truth

Covenant is 60% production-ready as a hackathon project. The circuits, contract architecture, and view key system are solid. The two make-or-break items are:

1. The full UltraHonk verifier — This is hard. Not "weekend project" hard. It requires deep understanding of the UltraHonk proof system, BN254 arithmetic in WASM, and the Protocol 26 host function semantics. The `rs-soroban-ultrahonk` reference implementation exists, but integrating it cleanly is the critical path.

2. Proving infrastructure — Institutions won't run `nargo` + `bb` locally. You need a proving API with sub-2-second latency, or a browser-based prover that doesn't require a 2GB download.

Everything else — nullifier pruning, view key rotation, regulator portals, multi-asset support — is straightforward engineering once the verifier and prover are production-grade.

If you solve the verifier + prover, the rest is execution.
